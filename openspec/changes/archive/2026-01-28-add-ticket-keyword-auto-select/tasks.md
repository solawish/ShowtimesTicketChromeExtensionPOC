## 2. 訂票按鈕與自動選票邏輯
- [x] 2.1 修改訂票按鈕啟用邏輯（例如 `updateBookButtonState()`）：當勾選框已勾選且「票種關鍵詞」有值（trim 後非空）時，不要求「票種選單已選擇」；其餘情況維持現有條件（須已選擇電影、場地、時間、票種、數量等）
- [x] 2.2 實作「依關鍵字與 fallback 規則選票」的純函數：輸入為票種列表（含 title）與關鍵字字串；輸出為符合規則的第一個票種或 null。規則順序：關鍵字包含 → 全票 → 單人套票 → 排除老人／愛心後取第一個
- [x] 2.3 在訂票按鈕點擊處理（例如 `handleBookClick()`）中：若勾選框已勾選且關鍵詞有值，則先呼叫票種 API 取得票種列表，再呼叫上述選票函數；若選出票種則將該票種設為當前選定票種後繼續既有訂票流程（取得座位、鎖位等）；若無符合票種則不繼續並顯示適當錯誤或提示
- [x] 2.4 若勾選框未勾選或關鍵詞為空，維持現有訂票流程（須已手動選擇票種，直接繼續取得座位等）
