# Change: 新增票種關鍵詞與自動選票邏輯

## Why
目前使用者必須手動從票種下拉選單選擇票種。當使用者希望依關鍵字或固定優先順序自動選票時（例如：先找關鍵字、再找全票／單人套票、或排除老人／愛心票），需要提供可選的自動選票機制與對應的 UI 設定，以提升購票流程的便利性。

## What Changes
- 在 popup 介面中，於「時間」與「票種」之間新增「票種關鍵詞」輸入框與一組勾選框（checkbox）
- **仍先讀取 API 列出票種**：選擇時間後照常呼叫票種 API，將票種填入下拉選單供參考
- **勾選時放寬訂票條件**：當勾選框勾選時，**忽視「票種 UI 必須已選擇」之限制**，訂票按鈕可在未手動選擇票種的情況下啟用（仍須已選擇電影、場地、時間等；啟用條件由實作定義，例如勾選且關鍵詞有值）
- **訂票按下後再選票**：當使用者按下「訂票」且勾選框已勾選時，**先呼叫票種 API**，再依下列順序自動選出票種，選出後再繼續後續訂票流程（取得座位、鎖位等）：
  1. 以關鍵字搜尋票種名稱（title）包含該關鍵字的票種，取第一個
  2. 若無：選擇名稱中含有「全票」的票種，取第一個
  3. 若無：選擇名稱中含有「單人套票」的票種，取第一個
  4. 若無：排除名稱中含有「老人」或「愛心」的票種後，取剩餘票種中的第一個
- 若自動選票無符合結果，則不繼續流程並給予適當提示
- 在 UI 上顯示勾選時的說明文字，讓使用者了解此選項的用途

## Impact
- Affected specs: ticket-configuration
- Affected code: `popup.html`, `popup.js`, `popup.css`（若需樣式）, Chrome Storage（若需持久化 checkbox 與關鍵詞）
